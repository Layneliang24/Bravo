# Bravo项目目录架构

## 📋 占位文件说明
- **`.keep`后缀文件**：占位文件，用于保持目录结构，可包含模板内容
- **无后缀文件**：实际业务文件，正式使用的代码文件
- **管理规则**：当创建实际文件时，可删除对应的.keep文件
- **检查方式**：通过.keep后缀快速识别哪些目录还需要实际文件

## 📋 .keep 后缀规范详解

### 🎯 使用规则

本项目采用 `.keep` 后缀来标识占位文件，便于区分模板文件和实际业务文件：

1. **占位文件标识**：所有模板文件和占位文件使用 `.keep` 后缀
2. **目录占位**：空目录使用 `directory.keep` 文件占位
3. **模板内容**：`.keep` 文件包含必要的模板代码和注释
4. **转换使用**：开发时可直接修改内容并去掉 `.keep` 后缀

### 🔄 管理流程

```bash
# 1. 检查占位文件
find . -name "*.keep" -type f

# 2. 转换为实际文件（示例）
mv users/models.py.keep users/models.py

# 3. 清理不需要的占位文件
./scripts/clean-keep-files.sh
```

### 📁 文件类型说明

- **`.py.keep`**：Python模块模板
- **`.ts.keep`**：TypeScript文件模板  
- **`.vue.keep`**：Vue组件模板
- **`.yaml.keep`**：配置文件模板
- **`.json.keep`**：数据文件模板
- **`directory.keep`**：目录占位文件

### ✅ 优势特点

- 🔍 **易于识别**：清晰区分占位文件和业务文件
- 🚀 **快速开发**：包含模板代码，修改后缀即可使用
- 🧹 **便于清理**：可批量删除不需要的占位文件
- 📊 **项目监控**：可统计实际业务文件的生成情况
